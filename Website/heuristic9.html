Page Not Found

When navigating through the Lewis School of Nursing and Health Professions subpage, the user will encounter Page Not Found pages. The site does not tell the user why the user encountered this error, nor does it give options for the user to navigate back to the page they were previously on. Another acceptable option would be to give links to sites that were related to the site the user originally intended to go to, yet all it gives the user is a search bar. In a way, this also violates the recognition rather than recall heuristic as well because it requires the user to remember what topic they were navigating through.

Failed Login

If an incorrect username/ password combination is input into PAWS, the site takes the user to a new page entirely that gives this error message. This is not the most efficient or constructive solution because it requires an extra step on the user’s part. This page is unnecessary because I would need to perform an extra action (‘Click Here’) to be able to re-enter my password. It would be better for them to simply make me stay on the same page and show an error message in red on the login page instead, like the next example shows.

This is an example of what a better error message looks like. On the Faculty and Staff login page, it doesn’t redirect the user to a new page and instead, displays the message in red underneath the login boxes. The user can then quickly re-enter a new username/ password combination.

Linked to Google Search

Because the site is linked to Google search, it automatically gives the user suggestions on spelling if they spelled a word wrong. This was a nice aspect because it helps the user identify and correct their mistake with a single click. However, to make it even better, the site should automatically give search results for the recommended spelling rather than not giving any results.
